## 🔴 Актуальная проблема (март 2026)

### Проблема: Зависание парсера и сбой цикла выполнения (28.02.2026 — 01.03.2026)

**Симптомы:**
- **28.02**: Парсер не запускается — ошибка `SessionNotCreatedException`
- **01.03**: Парсер зависает и не завершается за отведенные 360 секунд
- Последний успешный запуск: **27.02.2026** (данные записаны в Google Sheets, уведомление отправлено)

**Причины:**

1. **Headless режим** (включён 09.02 в коммите `389cd17`)
   - Вызывал `SessionNotCreatedException: Chrome instance exited` при наличии Lock-файла
   - Lock-файл остаётся от предыдущих сессий браузера в профиле пользователя
   - В headless режиме Selenium менее стабилен при работе с существующей сессией

2. **Проверка backdrop без таймаута** (добавлена 01.03 в коммите `cafb38b`)
   - Метод `_is_backdrop_active()` использовал `find_elements()` без явного таймаута
   - Полагался на `implicit_wait` (20 секунд по умолчанию)
   - 4 селектора × 20 сек = **80 секунд на одну проверку backdrop**
   - 5+ проверок за цикл = **400+ секунд** (таймаут задачи 360 сек)

## 🔴 Проблема 3: Низкая производительность проверки backdrop (03.03.2026)

**Симптомы:**
- Парсер работает медленно (~9-10 минут на выполнение)
- Большие паузы между операциями (до 80 секунд)

**Логи проблемы (03.03.2026, запуск в 14:04):**

**Анализ временных задержек:**

| Категория | Время | % от общего |
|-----------|-------|-------------|
| Полезная работа (навигация, извлечение данных) | ~2 мин | ~20% |
| Проверка оверлея (20 сек × 5 проверок) | ~100 сек | ~17% |
| **Проверка backdrop (80 сек × 5 проверок)** | **~400 сек** | **~67%** 🔴 |
| **ОБЩЕЕ ВРЕМЯ** | **~9 мин 20 сек** | **100%** |

**Причина:**

Метод `_is_backdrop_active()` использует `WebDriverWait` с таймаутом 1 секунда:

```python
# core/ozon_report_parser.py, строка ~615
backdrop_timeout = 1  # секунды на каждый селектор

for selector in backdrop_selectors:
    try:
        backdrop_elements = WebDriverWait(self.driver, backdrop_timeout).until(
            EC.presence_of_all_elements_located((By.XPATH, selector))
        )
```

**НО:** В методе `setup_browser()` установлен `implicit_wait(20)`:

```python
# core/base_parser.py, строка 234
timeout = config.get('timeout', self.config.get('DEFAULT_TIMEOUT', 60))
self.driver.implicitly_wait(timeout)  # timeout = 20 секунд по умолчанию
```

**Проблема:** В Selenium 4.x `WebDriverWait` и `implicit_wait` работают совместно через `PollingManager`. `WebDriverWait` ждёт до `max(implicit_wait, explicit_timeout)`.

**Результат:**
- `WebDriverWait(self.driver, 1)` ждёт **20 секунд** вместо 1
- 4 селектора × 20 секунд = **80 секунд на проверку backdrop**
- 5 проверок за цикл = **400 секунд (6.7 минут)**

**Влияние на другие методы:**

Метод `_is_overlay_present()` также использует `WebDriverWait` с таймаутом 5 секунд:

```python
# core/ozon_report_parser.py, строка ~520
wait_timeout = self.config.get("wait_timeout", 5)

elements = WebDriverWait(self.driver, wait_timeout).until(
    EC.presence_of_all_elements_located((By.XPATH, selector))
)
```

**Ожидалось:** 5 секунд
**Фактически:** 20 секунд (implicit_wait)

**Итоговая статистика потерь:**
- Проверка backdrop: 400 секунд
- Проверка оверлея: 75 секунд (15 секунд лишних на каждую из 5 проверок)
- **Общие потери: ~475 секунд (7.9 минут)**

**Решение:**

Временно уменьшать `implicit_wait` перед вызовом `WebDriverWait`:

```python
def _is_backdrop_active(self) -> bool:
    # ... код ...
    
    # Сохраняем текущий implicit_wait
    old_implicit_wait = self.driver.timeouts.implicit_wait
    
    try:
        # Устанавливаем короткий implicit_wait для быстрой проверки
        self.driver.implicitly_wait(0.5)  # 0.5 секунды вместо 20
        
        for selector in backdrop_selectors:
            try:
                backdrop_elements = WebDriverWait(self.driver, backdrop_timeout).until(
                    EC.presence_of_all_elements_located((By.XPATH, selector))
                )
                # ... проверка элементов ...
            except Exception:
                pass
    finally:
        # Восстанавливаем implicit_wait
        self.driver.implicitly_wait(old_implicit_wait)
```

**Ожидаемый результат после оптимизации:**
- Проверка backdrop: 4 селектора × 1 сек = **4 секунды**
- 5 проверок за цикл = **20 секунд** вместо 400
- **Экономия: 380 секунд (6.3 минут)**
- **Общее время выполнения: ~3 минуты** вместо 9-10 минут
- **Ускорение в 3 раза!**

**Статус:** ✅ **РЕАЛИЗОВАНО (03.03.2026)**

## 🔍 Диагностика проблемы с backdrop (март 2026)

```

21:34:24 DEBUG Проверка backdrop: 4 селекторов из конфигурации
```

**Причина:**

Метод `_is_backdrop_active()` использовал `find_elements()` без таймаута,
полагаясь на `implicit_wait` (20 секунд):

- 4 селектора × 20 сек = **80 секунд на одну проверку**
- 5+ проверок за цикл = **400+ секунд** (таймаут задачи 360 сек)

**Конфигурация backdrop_selectors:**

```python
"overlay_config": {
    "backdrop_selectors": [
        "//div[contains(@class, 'ozi__backdrop__backdrop__')]",
        "//div[contains(@class, 'ozi__backdrop')]",
        "//div[contains(@class, 'backdrop')]",
        "//div[contains(@class, 'modal-backdrop')]"
    ]
}
```

**Решение:**

Использован `WebDriverWait` с явным таймаутом 1 секунда:

```python
# Было (медленно):
backdrop_elements = self.driver.find_elements(By.XPATH, selector)

# Стало (быстро):
backdrop_timeout = 1  # секунды на каждый селектор
backdrop_elements = WebDriverWait(self.driver, backdrop_timeout).until(
    EC.presence_of_all_elements_located((By.XPATH, selector))
)
```

**Файл для исправления:**
`core/ozon_report_parser.py` — метод `_is_backdrop_active()`



**Причина:**

- Headless режим + Lock-файл профиля Edge = конфликт сессий
- Lock-файл остаётся от предыдущего незавершённого сеанса браузера
- В headless режиме браузер не может корректно обработать Lock-файл

**Решение:**

Отключить headless режим в конфигурации:

```python
# configs/base_configs/base_config.py
"browser_config": {
    "headless": False,  # Было: True
}
```

**Поиск в логах:**

```bash
# Поиск ошибок сессии
grep -r "SessionNotCreatedException" .\logs\reports_domain\Parser\

# Поиск Lock-файлов
grep -r "Файл блокировки" .\logs\reports_domain\Parser\
```
