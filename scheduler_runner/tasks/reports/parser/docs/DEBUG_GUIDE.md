# 🔧 Руководство по отладке парсера Ozon

## 📋 Обзор изменений

В рамках локализации проблемы периодических сбоев парсера добавлена расширенная система логирования и диагностики.

**Основная проблема:** Периодически парсер перенаправляется на страницу логина вместо целевых страниц отчетов.

**Причина:** Сессия браузера аннулируется (пользователь вышел вручную, истек таймаут сессии).

**Решение:** Добавлено детальное логирование для быстрой диагностики + скриншоты/HTML дампы при ошибках.

---

## 🎯 Новые возможности логирования

### 1. Диагностика перед навигацией

**Файл:** `core/base_report_parser.py`  
**Метод:** `navigate_to_target()`

**Что логируется:**

```
=== ДИАГНОСТИКА ПЕРЕД НАВИГАЦИЕЙ ===
Сессия драйвера: eb39d9d5d3...
Текущий URL перед переходом: https://turbo-pvz.ozon.ru/reports/giveout?...
Заголовок страницы перед переходом: Отчеты по выдачам
```

**Признак проблемы:**

```
⚠️ ВНИМАНИЕ: браузер уже находится на странице ЛОГИНА!
Возможная причина: сессия истекла или выполнен выход из системы
```

---

### 2. Диагностика после навигации

**Что логируется:**

```
=== ДИАГНОСТИКА ПОСЛЕ НАВИГАЦИИ ===
Фактический URL после перехода: https://turbo-pvz.ozon.ru/reports/giveout?...
Заголовок страницы: Отчеты по выдачам
```

**Критическая ошибка:**

```
❌ ПРОИЗОШЛО ПЕРЕНАПРАВЛЕНИЕ НА СТРАНИЦУ ЛОГИНА!
URL логина: https://turbo-pvz.ozon.ru/login?filter={...}&redirectUri=/reports/giveout?...
Перенаправление с целевой страницы: /reports/giveout?filter={...}
📸 Скриншот страницы логина сохранен: ./logs/reports_domain/Parser/error_login_20260226_213045.png
📄 HTML страницы сохранен: ./logs/reports_domain/Parser/error_login_20260226_213045.html
```

---

### 3. Диагностика страницы при проверке ПВЗ

**Файл:** `core/ozon_report_parser.py`  
**Метод:** `ensure_correct_pvz()`

**Что логируется:**

```
=== ДИАГНОСТИКА СТРАНИЦЫ ===
Текущий URL: https://turbo-pvz.ozon.ru/login?...
Заголовок страницы: Система управления заказами ПВЗ
❌ СТРАНИЦА ЛОГИНА ОБНАРУЖЕНА при проверке ПВЗ!
Сессия невалидна — возможна причина: выход пользователя или истек таймаут
Размер HTML страницы: 15234 байт
```

**Предупреждение:**

```
⚠️ ПВЗ не определен (Unknown) — возможна проблема с сессией или структурой страницы
Проверьте, что браузер находится на правильной странице, а не на странице логина
```

## 📁 Артефакты отладки

### Скриншоты ошибок

**Формат имени:**
- `error_login_YYYYMMDD_HHMMSS.png` — при перенаправлении на страницу логина
- `error_navigation_YYYYMMDD_HHMMSS.png` — при критических ошибках навигации
- `error_dropdown_YYYYMMDD_HHMMSS.png` — при проблеме с выбором опции в dropdown

**Расположение:** `./logs/reports_domain/Parser/`

**Когда создаются:**
- При обнаружении перенаправления на страницу логина
- При критических ошибках навигации
- При проблеме с выбором опции в dropdown (0 найдено опций)

> **Примечание:** Скриншоты создаются встроенным методом Selenium `save_screenshot()` — дополнительные пакеты не требуются!

---

### HTML дампы страниц

**Формат имени:** `error_login_YYYYMMDD_HHMMSS.html`

**Расположение:** `./logs/reports_domain/Parser/`

**Когда создаются:**
- При перенаправлении на страницу логина

**Зачем нужно:**
- Полный DOM страницы для анализа структуры
- Возможность открыть в браузере и изучить элементы
- Поиск изменений в CSS-классах, селекторах

**Симптомы в логе:**

```

21:30:40 DEBUG Оверлей не обнаружен
21:31:01 DEBUG Поиск элемента по селектору: //input[contains(@class, 'ozi__input__input__ie7wU') and @readonly]
21:31:21 DEBUG Ошибка при получении значения элемента ... no such element
21:31:21 DEBUG Сессия драйвера активна: eb39d9d5d3...
21:31:21 DEBUG Текущий URL: https://turbo-pvz.ozon.ru/login?...
21:31:21 DEBUG Заголовок страницы: Система управления заказами ПВЗ
```

**Причина:** Сессия аннулирована (пользователь вышел, истек таймаут)

**Решение:** Ручная авторизация в системе

---

### Проблема 2: Элемент ПВЗ не найден

**Симптомы в логе:**

```
21:30:40 DEBUG Требуемый ПВЗ: ЧЕБОКСАРЫ_340
21:30:40 DEBUG Используемый селектор для получения ПВЗ: //input[contains(@class, 'ozi__input__input__ie7wU') and @readonly]
21:31:01 DEBUG Ошибка при получении значения элемента ... no such element
21:31:01 DEBUG Текущий ПВЗ: Unknown
```

**Возможные причины:**
1. Страница логина вместо целевой (см. Проблему 1)
2. Изменились CSS-классы элементов
3. Структура страницы изменилась

**Диагностика:**
- Проверить `Текущий URL` — если содержит `login`, см. Проблему 1
- Открыть сохраненный HTML файл, проверить наличие элемента
- Сравнить селекторы с актуальной версией страницы

---

### Проблема 3: Оверлей не закрывается

**Симптомы в логе:**

```
21:30:20 DEBUG Проверка оверлея: selector=//div[contains(@class, 'ozi__dialog__dialog__C2BB8')], timeout=5s
21:30:40 DEBUG Оверлей не обнаружен: Message: timeout
```

**Возможные причины:**
1. Изменился CSS-класс оверлея
2. Это другой тип модального окна
3. Окно не является оверлеем (например, push-уведомление)

**Диагностика:**
- Открыть скриншот ошибки
- Изучить HTML на наличие модальных элементов
- Предложить новый селектор

## 🎓 Рекомендации по улучшению отладки

### 1. Включите подробное логирование

```bash
python scheduler_runner/tasks/reports/reports_processor.py --detailed_logs
```

### 2. Сохраняйте артефакты

Не удаляйте скриншоты и HTML файлы сразу — они могут понадобиться для анализа повторяющихся проблем.

### 3. Сравнивайте с успешными логами

При анализе проблемы сравнивайте логи неудачного запуска с успешными логами других объектов.

### 4. Документируйте находки

Если обнаружена новая причина сбоя, добавьте её в этот документ.

---

## 📞 Эскалация проблем

При обнаружении повторяющихся проблем:

1. Соберите логи за проблемный период
2. Приложите скриншоты и HTML дампы
3. Опишите контекст (время, действия пользователей)
4. Обратитесь к разработчику
