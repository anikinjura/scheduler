# Метод `_update_execution_date()`

## Версия
**0.0.1**

## Описание
Метод `_update_execution_date()` обновляет дату выполнения на основе аргументов командной строки, конфигурации или текущего времени. Метод определяет приоритетность источников даты и устанавливает соответствующее значение в конфигурации.

## Сигнатура
```python
def _update_execution_date(self)
```

## Приоритетность источников даты
1. Аргумент командной строки `--report_date` (наивысший приоритет)
2. Значение из конфигурации `execution_date`
3. Текущая дата (по умолчанию)

## Используемые параметры конфигурации
- **execution_date** (`str`): Дата выполнения, установленная в конфигурации
- **date_format** (`str`): Формат даты (по умолчанию '%Y-%m-%d')

## Изменения в версии 0.0.1
- Метод не изменен, но используется в контексте мульти-шаговой обработки
- Дата выполнения влияет на формирование URL, которые затем используются для создания `__STEP_SOURCE_URL__`
- `__STEP_SOURCE_URL__` из всех шагов используется для формирования `__SOURCE_URL__` как общего префикса всех URL с помощью метода `_get_common_url_prefix()`

## Примеры
- Если передан аргумент `--report_date 2023-12-25`, будет использована эта дата
- Если аргумент не передан, но в конфигурации есть `execution_date`, будет использовано это значение
- Если ни аргумент, ни конфигурация не содержат дату, будет использована текущая дата

## Особенности
- Метод вызывается в инициализации парсера
- Дата выполнения используется при формировании URL для навигации
- URL, сформированные с использованием даты выполнения, используются для создания `__STEP_SOURCE_URL__`
- `__STEP_SOURCE_URL__` из всех шагов используется для формирования `__SOURCE_URL__` как общего префикса всех URL с помощью метода `_get_common_url_prefix()`
