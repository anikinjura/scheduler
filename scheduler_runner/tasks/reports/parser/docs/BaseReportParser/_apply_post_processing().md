# Метод `_apply_post_processing()`

## Версия
**0.0.1**

## Описание
Метод `_apply_post_processing()` применяет постобработку к извлеченному значению. Метод может преобразовать тип значения, установить значение по умолчанию в случае ошибки и выполнить другие операции по обработке результата.

## Сигнатура
```python
def _apply_post_processing(self, value, post_processing_config)
```

## Параметры
- **value**: Извлеченное значение, которое нужно обработать
- **post_processing_config** (`dict`): Конфигурация постобработки, содержащая:
  - **convert_to** (`str`, optional): Тип для преобразования ('int', 'float', 'str', 'bool')
  - **default_value** (`any`, optional): Значение по умолчанию, которое будет возвращено в случае ошибки
  - **transform** (`str`, optional): Метод трансформации ('strip', 'upper', 'lower', 'title')

## Возвращаемое значение
Обработанное значение в соответствии с конфигурацией постобработки

## Изменения в версии 0.0.1
- Метод используется в `_extract_value_by_config()` для постобработки извлеченных значений
- Результаты постобработки могут быть частью результатов шагов, которые содержат `__STEP_SOURCE_URL__`
- `__STEP_SOURCE_URL__` из всех шагов используется для формирования `__SOURCE_URL__` как общего префикса всех URL с помощью метода `_get_common_url_prefix()`

## Пример конфигурации
```python
value = " 123 "
post_processing_config = {
    "convert_to": "int",
    "default_value": 0
}
# Результат: 123
```

## Особенности реализации
- Метод вызывается из `_extract_value_by_config()` для постобработки извлеченных значений
- Результаты постобработки могут быть частью результатов шагов, которые содержат `__STEP_SOURCE_URL__`
- `__STEP_SOURCE_URL__` из всех шагов используется для формирования `__SOURCE_URL__` как общего префикса всех URL с помощью метода `_get_common_url_prefix()`
