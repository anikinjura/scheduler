# Метод `_is_overlay_present()`

## Версия
**0.0.2**

## Описание
Метод `_is_overlay_present()` проверяет наличие оверлея (модального окна) на странице по заданному XPath-селектору. Метод ожидает появления элемента в DOM-дереве и проверяет его видимость.

## Сигнатура
```python
def _is_overlay_present(self, selector: str, timeout: int = 5) -> bool
```

## Параметры
- **selector** (`str`): XPath или CSS селектор для поиска оверлея
- **timeout** (`int`, optional): Время ожидания появления элемента в секундах (по умолчанию 5)

## Возвращаемое значение
- `True` — если оверлей найден в DOM-дереве и виден (`is_displayed()`)
- `False` — если оверлей не найден или скрыт

## Алгоритм работы

1. **Ожидание появления**: Использует `WebDriverWait` для ожидания появления элемента в DOM
2. **Проверка количества**: Если элементов не найдено — возвращает `False`
3. **Проверка видимости**: Проходит по всем найденным элементам и проверяет `is_displayed()`
4. **Возврат результата**: 
   - `True` — если хотя бы один элемент виден
   - `False` — если все элементы скрыты

## Логирование

Метод ведёт логирование:
- `TRACE` — вход в метод с селектором
- `DEBUG` — ожидание появления, результат проверки
- `INFO` — оверлей найден и виден (с количеством элементов)

## Пример использования

```python
# Проверка наличия оверлея
overlay_selector = "//div[contains(@class, 'ozi__dialog__dialog__C2BB8')]"
if self._is_overlay_present(overlay_selector, timeout=5):
    if self.logger:
        self.logger.info("Обнаружен оверлей, закрываем...")
    # Закрыть оверлей
else:
    if self.logger:
        self.logger.debug("Оверлей не обнаружен")
```

## Зависимости
- `selenium.webdriver.support.ui.WebDriverWait`
- `selenium.webdriver.support.expected_conditions`
- `selenium.webdriver.common.by.By`
