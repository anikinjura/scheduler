# Метод `_cleanup_lock_files()`

## Версия
**0.0.2**

## Описание
Метод `_cleanup_lock_files()` реализует логику удаления Lock-файлов из профиля браузера Edge. Lock-файлы создаются браузером при работе и могут оставаться после некорректного завершения, что блокирует повторное использование профиля.

## Сигнатура
```python
def _cleanup_lock_files(self, user_data_dir: str) -> None
```

## Параметры

| Параметр | Тип | Описание |
|----------|-----|----------|
| `user_data_dir` | `str` | Путь к директории пользовательских данных браузера (профиль Edge) |

## Возвращаемое значение
- `None` (метод ничего не возвращает)

## Какие файлы удаляются

Метод удаляет следующие Lock-файлы:

1. `Default\Lock` — основной Lock-файл профиля
2. `Default\LOCK` — альтернативный Lock-файл (верхний регистр)
3. `Default\SingletonLock` — Lock-файл одиночного экземпляра
4. `SingletonLock` — глобальный Lock-файл

## Алгоритм работы

1. **Формирование путей**: Создаёт полные пути к Lock-файлам на основе `user_data_dir`
2. **Проверка существования**: Для каждого файла проверяет наличие через `os.path.exists()`
3. **Удаление**: Если файл существует — удаляет через `os.remove()`
4. **Обработка ошибок**: Если удаление не удалось — логирует ошибку, продолжает работу

## Логирование

Метод ведёт логирование:
- `DEBUG` — информация о найденных и удалённых Lock-файлах
- `ERROR` — ошибки при удалении файлов

## Пример использования

```python
# В методе setup_browser() перед запуском браузера
user_data_dir = self._get_default_browser_user_data_dir()
self._cleanup_lock_files(user_data_dir)

# Создание драйвера после очистки Lock-файлов
options.add_argument(f"--user-data-dir={user_data_dir}")
driver = webdriver.Edge(options=options)
```

## Зависимости
- `os.path.exists()` — проверка существования файла
- `os.remove()` — удаление файла
- `user_data_dir` — путь к профилю браузера

## Обработка ошибок

Метод **не прерывает работу** при ошибках:
- Если файл не найден — продолжает работу
- Если файл не удаляется (занят) — логирует ошибку, продолжает работу
- Если нет прав доступа — логирует ошибку, продолжает работу

**Обоснование:** Lock-файлы могут быть удалены автоматически при завершении процессов браузера. Если удаление не удалось — браузер может завершиться успешно или завершить процессы самостоятельно.

## Пример диагностики

```
[10:32:18] DEBUG Очистка Lock-файлов в C:/Users/y.ank/AppData/Local/Microsoft/Edge/User Data...
[10:32:18] DEBUG Удалён Lock-файл: C:/Users/y.ank/AppData/Local/Microsoft/Edge/User Data\Default\Lock
[10:32:18] DEBUG Удалён Lock-файл: C:/Users/y.ank/AppData/Local/Microsoft/Edge/User Data\SingletonLock
```

## Типичные проблемы

### Ошибка: "Файл занят другим процессом"

**Причина:** Браузер или процесс Edge всё ещё использует Lock-файл.

**Решение:**
1. Завершить процессы браузера через `_terminate_browser_processes()`
2. Подождать 2-3 секунды
3. Повторить `_cleanup_lock_files()`

### Ошибка: "Отказано в доступе"

**Причина:** Недостаточно прав для удаления файлов.

**Решение:**
1. Запустить задачу от имени администратора
2. Проверить права на папку профиля
3. Убедиться, что все процессы Edge завершены
